# v1.2 Multi-Vendor Preset Ergonomics — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v1.2 of `llm-vision-mcp` with preset-based ergonomics for 8 OpenAI-compatible vision vendors (moonshot, zai, qwen, nvidia, groq, together, deepinfra, xai). A user who names a preset via `VISION_DEFAULT_PROVIDER` and sets the vendor's standard API key env var gets a working provider without writing any JSON.

**Architecture:** Add a pure-data preset table in `src/presets.ts`. Add a single new helper `applyPresetDefaults()` at the end of `loadConfig` that synthesizes an OpenAI-compatible provider entry for preset-named default providers, filling gaps from preset defaults → vendor env vars → CLI flags → config file (file-entry fields always win when present). Add generic `--api-key` / `--base-url` CLI flags that scale to N presets without per-vendor clutter. Add a startup validation that fails fast when `defaultProvider` doesn't resolve to any registered provider.

**Tech Stack:** TypeScript (strict), vitest, `@modelcontextprotocol/sdk`, OpenAI SDK (via the existing `OpenAICompatibleProvider` subclass). TDD throughout. Commit after each task.

**Spec reference:** `docs/superpowers/specs/2026-04-21-v1.2-multi-vendor-presets-design.md`.

**Non-goals (out of scope):** Auto-scanning preset env vars at startup; multi-preset activation via env vars only; new provider classes for non-OpenAI-compatible vendors like Minimax; per-preset retry/concurrency tuning. These are all explicitly deferred in the spec.

- [ ] **Step 3: Implement the helper and wire it into `loadConfig`**

At the top of `src/config.ts`, add an import:

```ts
import { PRESETS } from "./presets.js";
```

Add the helper near the end of the file, before `loadConfig` closes:

```ts
function applyPresetDefaults(config: AppConfig): void {
  const name = config.defaultProvider;
  const preset = PRESETS[name];
  if (!preset) return;
  const existing = config.providers[name];
  // Respect an explicit non-OpenAI-compatible type set via config file.
  if (existing?.type && existing.type !== "openai-compatible") return;
  const apiKey = existing?.apiKey ?? process.env[preset.envKey];
  if (!apiKey) return;
  config.providers[name] = {
    type: "openai-compatible",
    apiKey,
    baseUrl: existing?.baseUrl ?? process.env[preset.envBaseUrl] ?? preset.baseUrl,
    model: existing?.model ?? process.env[preset.envModel] ?? preset.defaultModel,
    ...(existing?.timeout !== undefined ? { timeout: existing.timeout } : {}),
    ...(existing?.retry !== undefined ? { retry: existing.retry } : {}),
    ...(existing?.concurrency !== undefined ? { concurrency: existing.concurrency } : {}),
  };
}
```

In `loadConfig`, add a call to the helper as the very last step before `return config;`:

```ts
  applyPresetDefaults(config);

  return config;
```

```ts
const BUILTINS = ["openai", "anthropic", "google", "ollama"];

export function validateDefaultProvider(
  defaultProvider: string,
  registered: string[],
): void {
  if (registered.includes(defaultProvider)) return;
  const presetNames = Object.keys(PRESETS).join(", ");
  const builtinNames = BUILTINS.join(", ");
  throw new Error(

      `  Known builtins: ${builtinNames}\n` +
      `  Or provide a vision-config.json entry for "${defaultProvider}".`,
  );
}
```

```ts
    process.exit(1);
  }
```
